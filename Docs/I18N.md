# KLineLens — i18n / Language

> Language and text strategy.

---

## MVP Decision: English Only

For MVP, the application uses **English only**. This simplifies:
- Backend output (no key mapping needed)
- Frontend rendering (no translation layer)
- Initial development speed

Multi-language support (Chinese) is planned for post-MVP backlog.

---

## Future i18n Strategy (Post-MVP)

When adding multi-language support:

### 1. Backend Output Strategy

Backend should output **keys** for user-facing text:

```json
{
  "behavior": {
    "dominant": "shakeout",
    "evidence": [
      {
        "note_key": "broke_support_then_reclaimed",
        "metrics": {...}
      }
    ]
  }
}
```

Frontend maps keys to localized text based on user language preference.

### 2. Text Keys (Reserved)

UI Keys:
- `search_placeholder`
- `market_state`
- `behavior`
- `evidence`
- `timeline`
- `playbook`
- `loading`
- `error_no_data`
- `error_rate_limited`

Reason Keys:
- `support_sweep_reclaim`
- `volume_spike_no_progress_up`
- `breakout_confirmed`
- `breakout_failed_fakeout`

Risk Keys:
- `noise_high_1m`
- `liquidity_risk`
- `event_risk`

### 3. Storage

```
localStorage key: kline_lens_lang
values: "en" | "zh"
default: "en"
```

### 4. Implementation

```typescript
// Future: apps/web/src/lib/i18n.ts
const translations = {
  en: {
    search_placeholder: "Enter ticker symbol...",
    market_state: "Market State",
    // ...
  },
  zh: {
    search_placeholder: "输入股票代码...",
    market_state: "市场状态",
    // ...
  }
};

export function t(key: string): string {
  const lang = localStorage.getItem('kline_lens_lang') || 'en';
  return translations[lang][key] || key;
}
```

## Current MVP Implementation

For MVP, text is hardcoded in English:

```typescript
// MVP: Direct English text
<h2>Market State</h2>
<p>Regime: {report.market_state.regime}</p>
```

No translation layer needed for MVP.
